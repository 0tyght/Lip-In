# คู่มือใช้งานจริง Lip In Money

เอกสารนี้คือวิธีใช้แอปกับเงินจริง ตั้งแต่ติดตั้งบน iPhone ไปจนถึงการนำเข้า statement และเชื่อมต่อธนาคารผ่าน backend ส่วนตัว

## 1. เข้าแอปจากที่ไหน

ใช้ URL GitHub Pages:

```text
https://0tyght.github.io/Lip-In/
```

ถ้าเปิดจากคอมเพื่อทดสอบในเครื่อง:

```powershell
cd "C:\xampp\htdocs\Lip In"
node dev-server.mjs
```

แล้วเปิด:

```text
http://127.0.0.1:4173/
```

ถ้า port 4173 ถูกใช้อยู่:

```powershell
$env:PORT=4174
node dev-server.mjs
```

## 2. ติดตั้งบน iPhone แบบ PWA

1. เปิด Safari บน iPhone
2. เข้า `https://0tyght.github.io/Lip-In/`
3. กดปุ่ม Share
4. เลือก `Add to Home Screen`
5. ตั้งชื่อ `Lip In`
6. เปิดแอปจากไอคอนบน Home Screen

หลังติดตั้งแล้ว แอปจะทำงานแบบ PWA และโหลดหน้าจอพื้นฐานได้แม้อินเทอร์เน็ตไม่เสถียร ส่วนการซิงก์ธนาคารและการอัปเดตเวอร์ชันต้องใช้อินเทอร์เน็ต

## 3. เริ่มใช้ครั้งแรก

แอปเริ่มด้วยข้อมูลว่างสำหรับเงินจริง:

- มี `บัญชีหลัก` และ `เงินสด` ยอด 0 บาท
- ไม่มีรายการธุรกรรม
- ไม่มีงบประมาณ
- ไม่มีเป้าหมาย
- ไม่มีรายการผ่อนชำระ

ให้ตั้งค่าพื้นฐานตามนี้:

1. ไปที่ `กระเป๋า`
2. แก้ชื่อกระเป๋าให้ตรงกับชีวิตจริง เช่น `KBank`, `SCB`, `เงินสด`, `บัตรเครดิต`
3. ใส่ยอดคงเหลือจริง ณ วันที่เริ่มใช้
4. เพิ่มกระเป๋าใหม่ถ้ามีหลายบัญชี
5. ถ้าเป็นบัตรเครดิตหรือหนี้ ให้ตั้งเป็นกระเป๋าหนี้สิน/ยอดติดลบตามรูปแบบที่ใช้ติดตาม

## 4. บันทึกรายการจริงรายวัน

กดปุ่ม `+` ตรงกลางล่าง แล้วเลือกประเภท:

- `รายจ่าย`: เงินออก เช่น อาหาร เดินทาง ค่าไฟ
- `รายรับ`: เงินเข้า เช่น เงินเดือน รับโอน
- `โอนเงิน`: ย้ายเงินจากกระเป๋าหนึ่งไปอีกกระเป๋า
- `ผ่อนชำระ`: ใช้กับรายการที่เกี่ยวกับหนี้/ผ่อน

ฟิลด์ที่ควรใส่ทุกครั้ง:

- ชื่อรายการ
- จำนวนเงิน
- วันที่และเวลา
- หมวดหมู่
- กระเป๋าต้นทาง
- ร้านค้า/ผู้รับเงิน ถ้ามี
- แท็ก เช่น `บ้าน`, `งาน`, `ทริป`
- หลักฐานแนบ เช่น เลขอ้างอิง รายการโอน หรือชื่อไฟล์ใบเสร็จ

ถ้าเป็นรายการที่ยังไม่ตัดบัญชี ให้เลือกสถานะ `รอตรวจสอบ` หรือ `ตั้งเวลาล่วงหน้า` เพื่อไม่ให้ยอดกระเป๋าเพี้ยนก่อนรายการเกิดจริง

## 5. ตั้งงบประมาณ

ไปที่ `งบประมาณ` แล้วกด `+ เพิ่ม`

ตั้งงบตามหมวดที่ใช้จริง เช่น:

- อาหาร
- เดินทาง
- กาแฟ
- ค่าไฟ
- ช้อปปิ้ง
- สุขภาพ

หน้ารายงานจะแสดง `งบประมาณ vs ใช้จริง` เป็นกราฟแท่งแนวนอน เพื่อดูว่าหมวดไหนใกล้เกินงบ

## 6. ตั้งเป้าหมายออมเงิน

ไปที่ `เป้าหมาย`

ใช้กับสิ่งที่ต้องเก็บเงิน เช่น:

- เงินสำรองฉุกเฉิน
- เที่ยว
- ซื้อของใหญ่
- ลงทุน

แอปจะแสดง progress เพื่อดูว่าถึงเป้าหมายกี่เปอร์เซ็นต์แล้ว

## 7. ตั้งรายการผ่อนชำระ

ไปที่ `ผ่อนชำระ` แล้วเพิ่มรายการ เช่น:

- ผ่อนบ้าน
- ผ่อนรถ
- ผ่อนบัตร
- สินเชื่อ

ใส่เงินต้น อัตราดอกเบี้ย จำนวนงวด และยอดจ่ายงวดถัดไป แอปจะช่วยติดตามเงินต้น ดอกเบี้ย และงวดที่จ่ายแล้ว

## 8. นำเข้า statement CSV

วิธีนี้ใช้งานจริงได้ทันทีโดยไม่ต้องมี Bank API

1. เข้าแอปธนาคารหรือเว็บธนาคาร
2. ดาวน์โหลด statement เป็น CSV
3. เปิด Lip In
4. ไปที่ `เมนู` หรือ `กระเป๋า`
5. กด `ธนาคาร`
6. กด `นำเข้า statement`
7. เลือกไฟล์ CSV

คอลัมน์ที่รองรับ:

- วันที่: `date`, `transaction date`, `วันที่`
- รายละเอียด: `description`, `details`, `รายการ`, `รายละเอียด`
- จำนวนเงิน: `amount`, `debit`, `credit`, `เงินออก`, `เงินเข้า`
- เลขอ้างอิง: `reference`, `transaction id`, `เลขที่รายการ`

แอปจะกันรายการซ้ำด้วยเลขอ้างอิง หรือ hash จากวันที่ รายละเอียด จำนวนเงิน และบัญชี

## 9. เชื่อมธนาคารอัตโนมัติแบบจริง

การซิงก์ธนาคารอัตโนมัติต้องมี backend เพราะ PWA ที่อยู่บน GitHub Pages ห้ามเก็บ secret ของธนาคารหรือ provider ไว้ในเครื่องผู้ใช้

โครงสร้างที่ใช้จริง:

```text
iPhone PWA -> Cloudflare Worker ของคุณ -> Plaid/Open Banking provider -> ธนาคาร
```

สิ่งที่ต้องมี:

- Cloudflare account
- Wrangler CLI
- KV namespace สำหรับเก็บ bank access token ฝั่ง Worker
- Provider credentials เช่น Plaid `client_id` และ `secret`
- `SYNC_API_TOKEN` ที่คุณตั้งเองเพื่อให้แอปเรียก Worker ได้

ขั้นตอนย่อ:

```powershell
cd "C:\xampp\htdocs\Lip In\workers"
copy wrangler.toml.example wrangler.toml
wrangler kv namespace create BANK_ITEMS
wrangler secret put PLAID_CLIENT_ID
wrangler secret put PLAID_SECRET
wrangler secret put SYNC_API_TOKEN
wrangler deploy
```

จากนั้นเปิด Lip In:

1. ไปที่ `ธนาคาร`
2. ใส่ `Bank API URL` เป็น URL ของ Worker
3. ใส่ `User ID` เช่น `lipin-personal`
4. ใส่ `Sync API token` ให้ตรงกับ Worker secret
5. กด `ตรวจ backend`
6. กด `เชื่อมธนาคารจริง`
7. กด `ซิงก์รายการจริง`

ข้อสำคัญ: ถ้าธนาคารของคุณไม่มี API หรือ provider ยังไม่รองรับประเทศ/ธนาคารนั้น ให้ใช้ CSV statement แทน หรือเปลี่ยน Worker connector ไปใช้ API ที่คุณได้รับสิทธิ์จากธนาคารโดยตรง

## 10. สำรองข้อมูล

ไปที่ `เมนู`

- `สำรอง JSON`: เก็บข้อมูลทั้งหมดของแอป
- `ส่งออก CSV`: เก็บรายการธุรกรรมไว้เปิดใน Excel/Google Sheets

ควรสำรอง JSON เป็นระยะ เพราะข้อมูลหลักของ PWA เก็บในเครื่องผ่าน localStorage หาก iOS เคลียร์พื้นที่หรือคุณลบเว็บข้อมูล ข้อมูลในเครื่องอาจหายได้

## 11. อัปเดตเวอร์ชัน

เมื่อเปิดแอปและมีเวอร์ชันใหม่ Service Worker จะดึงไฟล์ล่าสุดและสลับ cache ให้อัตโนมัติ หลัง reload จะได้เวอร์ชันใหม่ ไม่ต้องลบและติดตั้งใหม่

ถ้าอยากตรวจเอง:

1. ไปที่ `เมนู`
2. กด `ตรวจเวอร์ชันล่าสุด`
3. ถ้ามีเวอร์ชันใหม่ ให้ปิดเปิดแอปอีกครั้ง

## 12. วิธีใช้ประจำวันที่แนะนำ

เช้า:

- ดูยอดคงเหลือ
- ดูงบรายวัน
- ตรวจรายการที่รอตรวจสอบ

ระหว่างวัน:

- กด `+` ทุกครั้งที่จ่ายเงินหรือรับเงิน
- ถ้ามีใบเสร็จ ให้ใส่เลขอ้างอิง/ชื่อไฟล์ในช่องหลักฐานแนบ
- ถ้าโอนระหว่างบัญชี ให้ใช้ประเภท `โอนเงิน` เพื่อไม่ให้รายรับรายจ่ายเพี้ยน

สิ้นวัน:

- เช็กรายการล่าสุด
- นำเข้า statement ถ้ามี
- สำรอง CSV/JSON เมื่อมีข้อมูลสำคัญ

สิ้นเดือน:

- ดูรายงานหมวดหมู่
- ดูงบประมาณ vs ใช้จริง
- ดู cash flow
- ปรับงบเดือนถัดไป
